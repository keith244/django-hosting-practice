# Django Hosting Practice
---

## This is my hosting practice project
Recently I was able to complete a personal project. It was a very good experience. It was truly satisfying. However, the happiness soon faded as i realised that i couldn't host it, or rather, I didn't know how to host it. That meant that it would remain in my PC and GitHub forever - but not to be used by others.  

So I came up with this plan to learn how to host Django projects. 

---
## Why Django
I chose Django because, its my main choice of framework. I am most comfortable with Django and believe that im very profecient in it. Thus, I wanted to use it. 

---

## What the project contains
The project has 4 sub-projects(folders) inside it:
- basic_host -> Understand deployment basics
- styled_hosting -> Handling of static files
- host_dependencies -> Manage project dependencies
- host_with_dB -> Full-production-ready setup

---
#### What each is:

1. **basic_host**
A basic "Hello World" Django project. 
>It has no styling, one page, and only 2 sentences "Hello World"
>This folder, helps me to learn the basics of hosting. There'll be no styling at all and no database at all. Its just to help me learn what goes into hosting. 

2. **styled_hosting**
>This one is to help me learn how the server might interact with static files i.e. CSS, JS and even images. I want to know how deploy a site with static files. 
>I think this is very important because eventually, all my projects need to be styled if anyone's going to interact with them.
>I also believe its the next step after basic hosting with no styles.

3. **host_dependencies**
>This is to help me host a django project with multiple dependencies i.e. pillow, pymysql and any other common dependencies i might come across. It's a step from styled hosting as i also need to know how to handle packages, libraries and anything that my app requires to run. 

4. **host_with_dB**
>this i believe will be the ultimate challenge, hence i made it the last. The project will be a full Django app with dependencies, static files, and a database to store real data and will allow for basic crud processes. This i believe will be very important as that is where most of my projects are headed.
